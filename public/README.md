# Инструкция по добавлению изображений и иконок

## 📁 Структура папки public

Все статические файлы (изображения, иконки, логотипы) должны находиться в папке `public/`. Vite автоматически копирует их в корень собранного проекта.

## 🔧 Как заменить placeholder'ы на реальные изображения

1. Подготовьте изображения в соответствии с указанными размерами
2. Сохраните их с точными названиями файлов (как указано выше)
3. Скопируйте файлы в папку `public/`
4. Перезапустите dev-сервер (если он запущен)

## ⚙️ Как это работает

Vite автоматически обрабатывает файлы из папки `public/`:
- В коде используются пути типа `/icon.svg`
- Vite копирует файлы из `public/` в корень собранного проекта
- В production эти файлы будут доступны по тем же путям

## 📝 Примеры использования в коде

```tsx
// SVG иконка
<img src="/icon.svg" alt="Play" />

// PNG изображение
<img src="/imagewithfallback.png" alt="Background" />

// Логотип
<img src="/hacksaw-gaming-logo-0.svg" alt="Hacksaw Gaming" />
```

## 🎨 Рекомендации по изображениям

- **Формат**: Используйте SVG для иконок и логотипов, PNG/JPG для фотографий
- **Оптимизация**: Сжимайте изображения перед добавлением (используйте TinyPNG, ImageOptim и т.д.)
- **Размеры**: Указанные размеры являются рекомендуемыми для оптимальной производительности
- **Naming**: Сохраняйте точные названия файлов (с учетом регистра)

Сейчас в проекте используются простые placeholder'ы (минимальные SVG и пустые файлы). Замените их на реальные изображения для полноценного отображения дизайна.
